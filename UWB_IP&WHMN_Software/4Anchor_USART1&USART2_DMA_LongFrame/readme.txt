Document History
【2019.06.30】
1、串口1作用是接收上位机的命令，命令的形式是单个字符，采用阻塞方式，计划改为中断方式，不然串口2即使采用DMA方式，也会丢数据。

2、串口2采用DMA发送和接收，效果还可以，这样串口在接收数据时，就不会进入串口中断，也就不会影响测距了。如果采用串口中断方式的话，接口串口数据时，会频繁的进入中断，
   从而使得测距过程不连续，程序就容易卡死。

【2019.07.08】
1、串口1更改为DMA方式收发，普通测试没问题。
   【问题】：在UWB的接收回调函数中，使用串口阻塞方式发送UWB接收到的数据，可以完全发送，使用DMA方式的话，就会有一大段0，但是字节数是一样的。
   【解决方法】：采用阻塞方式发送，但是重新调整程序，发送消息的时候不定位。定位的时候不发送消息。
                新添的包类型定义：POLL_MSG\ANSWER_MSG
   【进度】：完成
2、DW1000的收发模式改为长帧模式（1023字节）。因为短帧模式每次只能发送127字节，但是生理参数信息每次很多，而且是多路PPG/ECG/IMU，所以127字节根本不够。
   【进度】：完成
